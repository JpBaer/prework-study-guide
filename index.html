HTML

  • The head elements contains metadata about the webpage
  • The body contains all of the content actually seen on the webpage
  • Each section has an id and class tag that are used for referencing with CSS and JS
  • headers vary in size starting from h1 all the way to h6
  • elements nested in a different "parent element" are called child elements
  • html creates the framework for a webpage
  • All code starts with a declaration !Doctype html
  • Next we have html tag often followed by language declaration (lang = "en")
  • Common elements of the head include:
    • title: Title of page for SEO
    • style: Defines CSS styles associated with site
    • base: Defines base URL
    • link: Connects external resources to html
    • meta: Defines metadata such as
      • Character set
      • Description
      • Keywords
      • Author
      • Viewpoint

CSS

  • A margin indicates how much space we want around the element
  • A padding indicates how much space we want around the content inside the element
  • There are three ways to style a webpage
    • inline: style an element directly in html file
    • internal: contains CSS rules in head section of html file
    • external: CSS rules in a seperate linked file
      • Use link element to link CSS sheet to html
      • most common method
  • CSS rules stsrt with a selector and then a declaration
    • Selector: Defines html elements to which declarations apply
    • Declarations start with the css property to apply followed by the value
  • You can combine two selecotrs into a single CSS rule with a comma
  • * is a selector that refers to all content on a page
  • Class Attribute allow us to share a CSS rule to any element without applying it to all elements
  • ."word" designqtes class selector, ."word" can then be referenced in html to apply the styling

Git

  • git status: checks what branch we are currently on
  • git checkout -b "branch name": creates a new working branch
  • git checkout main: returns you to the main branch
    • All changes need to be either commited or abandoned before returning to main branch
  • git add -A: Adds the changed files to working area of git
  • git commit -m "Change notes": Commits the changes to our repo with a descriptive label
  • It's a good habit to always pull your base branch into your feature branch before pushing any code
  • git pull origin main: syncs local branch with the remote branch in github
  • git push origin "branch name": pushes all of your changes
